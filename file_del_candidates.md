This audit is the deliverable for `forward_plan.md` §2.2 step 8. Every file in the repo (excluding `.git/`) was inspected. Files broken out below are those that look redundant, superseded, or orphaned.

## Confidence definitions

- **High** — file is provably orphaned (no live consumer) **or** explicitly slated for deletion by an existing plan; deleting it has no functional impact and the owner only needs to decide whether to keep it for historical reference.
- **Medium** — file is unreferenced by runtime code but has historical / supporting value; deletion is a judgement call (e.g. regenerable from another file in-tree, or a one-off artefact of completed work).
- **Low** — file is referenced somewhere or has plausible ongoing use; included for transparency because the candidate category in §2.2 invited scrutiny, but I'd lean keep.
